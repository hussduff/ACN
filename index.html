Networking Assignment Questions

2 Mark Questions

Explain the role of an Internet Service Provider (ISP).

An Internet Service Provider (ISP) is a company that provides you with an internet connection. Think of them as the company that gives you the special wire or signal to access the internet. They allow you to browse websites, send emails, and stream videos.

Expand ICANN and state its function.

ICANN stands for the Internet Corporation for Assigned Names and Numbers. Its job is to keep the internet organized. It makes sure that every website name and IP address is unique, so there are no mix-ups when you try to visit a website.

Explain the default subnet mask?

A subnet mask is like a special filter that works with an IP address. It tells a computer which part of the IP address is for the network and which part is for the specific device (like your computer or phone). The "default" mask is the standard one used for different types of IP addresses.

Define Subnet and supernet.

A subnet is a small network that is part of a larger one. The process of dividing a big network into smaller, more manageable parts is called subnetting. A supernet is the opposite. It's when you combine several small networks into one big network. The process of combining them is called supernetting.

4 Mark Questions

State the importance of IPv6 over IPv4.

IPv6 is an important evolution of the internet protocol due to several key advantages over IPv4:

  • Larger Address Space: IPv6 has 128-bit addresses, offering a practically limitless number of unique addresses ($2^{128}$), solving the address exhaustion problem of IPv4.
  • No NAT Required: Because of the vast address space, Network Address Translation (NAT) is no longer needed, simplifying network design and improving end-to-end connectivity.
  • Built-in Security: IPv6 includes built-in security features through IPsec, making it more secure by default.
  • More Efficient Routing: The IPv6 header is simpler and has a fixed size, which allows routers to process packets more quickly.
  • Improved Quality of Service (QoS): The "Flow Label" field in the IPv6 header allows for better handling of real-time data like video and audio streaming.
  • Auto-Configuration: IPv6 supports stateless address auto-configuration (SLAAC), which lets devices automatically get an IP address without a DHCP server, simplifying network administration.
  • Efficient Multicast: IPv6 uses multicast addresses more efficiently to send a single packet to multiple destinations at once, reducing network traffic.
  • No Broadcasts: IPv6 does not use broadcast addresses, which can be a source of network congestion. It replaces them with multicast for more targeted communication.
  • Improved Header Structure: The streamlined header removes fields that were rarely used in IPv4, making it more efficient.
  • Future-Proofing: The design of IPv6 ensures that the internet can continue to grow and support a massive number of new devices, like those in the Internet of Things (IoT).

Draw and explain the IPv4 header format.

The IPv4 header is like the envelope for an internet packet. It contains important information that helps the packet get to its destination. 

+----------------------+----------------------+----------------------+----------------------+
| Version (4) | IHL (4) | Type of Service (8) | Total Length (16) |
+----------------------+----------------------+----------------------+----------------------+
| Identification (16) | Flags (3) | Fragment Offset (13) |
+----------------------+----------------------+----------------------+----------------------+
| Time to Live (8) | Protocol (8) | Header Checksum (16) |
+----------------------+----------------------+----------------------+----------------------+
| Source IP Address (32) |
+----------------------+----------------------+----------------------+----------------------+
| Destination IP Address (32) |
+----------------------+----------------------+----------------------+----------------------+
| Options (variable) |
+----------------------+----------------------+----------------------+----------------------+

The key fields are:

  • 1. Version (4 bits): Indicates the IP version, which is 4.
  • 2. IHL (Internet Header Length, 4 bits): Specifies the size of the header, in 32-bit chunks. The minimum value is 5, meaning a 20-byte header.
  • 3. Type of Service (8 bits): Used for Quality of Service (QoS), allowing packets to be prioritized based on network needs (e.g., speed, reliability).
  • 4. Total Length (16 bits): The total size of the IP packet (header and data) in bytes. The maximum size is 65,535 bytes.
  • 5. Identification (16 bits): A unique ID for each packet sent from a host. All fragments of a single packet share the same ID.
  • 6. Flags (3 bits): Control how fragmentation happens. One bit indicates if the packet can be fragmented at all, and another indicates if there are more fragments to follow.
  • 7. Fragment Offset (13 bits): Specifies the position of a fragment in the original packet, helping the receiver reassemble the data in the correct order.
  • 8. Time to Live (TTL, 8 bits): A countdown timer. Each router the packet passes through decreases this number by one. If it hits zero, the packet is discarded to prevent infinite loops.
  • 9. Protocol (8 bits): Identifies the next-level protocol carried in the packet's data portion (e.g., TCP or UDP).
  • 10. Header Checksum (16 bits): A check to ensure the header hasn't been corrupted in transit. It is recalculated at each router.
  • 11. Source IP Address (32 bits): The IP address of the device that sent the packet.
  • 12. Destination IP Address (32 bits): The IP address of the intended recipient.

Draw and explain the IPv6 header format.

The IPv6 header is also like an envelope, but it's simpler and more efficient than the IPv4 one. It has a fixed size, which makes it easier for routers to process. 

+----------------------+----------------------+----------------------+
| Version (4) | Traffic Class (8) | Flow Label (20) |
+----------------------+----------------------+----------------------+
| Payload Length (16) | Next Header (8) | Hop Limit (8) |
+----------------------+----------------------+----------------------+
| Source Address (128) |
| ... |
| ... |
| ... |
+----------------------+----------------------+----------------------+
| Destination Address (128) |
| ... |
| ... |
| ... |
+----------------------+----------------------+----------------------+

The key fields are:

  • 1. Version (4 bits): Indicates the IP version, which is always 6.
  • 2. Traffic Class (8 bits): Similar to IPv4's Type of Service, it prioritizes packets for Quality of Service (QoS).
  • 3. Flow Label (20 bits): Used to label sequences of packets that need special handling, like real-time streaming, without needing to check all header details at each router.
  • 4. Payload Length (16 bits): The total size of the data and any extension headers that follow the main IPv6 header.
  • 5. Next Header (8 bits): Identifies the next header in the chain, which could be an extension header or the transport layer protocol (like TCP).
  • 6. Hop Limit (8 bits): The maximum number of hops a packet can take. It works like IPv4's TTL, preventing packets from looping forever.
  • 7. Source Address (128 bits): The full IPv6 address of the device that sent the packet.
  • 8. Destination Address (128 bits): The full IPv6 address of the intended recipient.
  • 9. No Checksum: Unlike IPv4, IPv6 does not have a header checksum, which speeds up processing at each router.
  • 10. Extension Headers: IPv6 uses optional extension headers to add extra information (like security or fragmentation details), which are placed between the main header and the payload. This modular design keeps the main header small and efficient.

Explain ARP working in brief.

ARP (Address Resolution Protocol) is a way for a device to find the physical address (MAC address) of another device on the same local network when it only knows the IP address. Here is how the process works:

  • 1. Check the Cache: When Device A wants to send data to Device B, it first checks its own ARP cache (a table of recent IP-to-MAC address mappings) to see if it already knows Device B's MAC address.
  • 2. Send a Broadcast: If the address isn't in the cache, Device A creates an ARP request message. This message asks, "Who has this IP address?" and includes Device A's own IP and MAC addresses.
  • 3. The Broadcast Spreads: The ARP request is sent as a broadcast, meaning every device on the local network receives it.
  • 4. The Target Replies: All devices check if the IP address in the request matches their own. Only the device that matches (Device B) will respond.
  • 5. Send a Unicast Reply: Device B creates an ARP reply message. This message contains its own MAC address and sends it directly back to Device A (a "unicast" message, not a broadcast).
  • 6. Update the Cache: Device A receives the reply and adds Device B's IP and MAC address to its ARP cache. This mapping is stored for a short time so the process doesn't have to repeat for every single packet.
  • 7. Final Communication: Now that Device A knows Device B's MAC address, it can send the data packet directly to it.
  • 8. Cache Cleanup: ARP cache entries have a lifespan. They expire after a certain amount of time to ensure the addresses are up-to-date.
  • 9. Proactive Updates: A device can also send a "gratuitous ARP" message to announce its presence and update other devices' caches without being asked.
  • 10. The IP-to-MAC Link: In essence, ARP is the fundamental process that links a logical IP address to a physical MAC address on a local network, allowing devices to communicate.

Explain ARP message format.

An ARP message is a simple message with a few key pieces of information, formatted into specific fields.

  • 1. Hardware Type (2 bytes): Specifies the type of network hardware, such as Ethernet.
  • 2. Protocol Type (2 bytes): Specifies the network protocol, such as IPv4.
  • 3. Hardware Address Length (1 byte): The length of the hardware address (e.g., 6 bytes for an Ethernet MAC address).
  • 4. Protocol Address Length (1 byte): The length of the protocol address (e.g., 4 bytes for an IPv4 address).
  • 5. Operation (2 bytes): Identifies if the message is an ARP request (1) or a reply (2).
  • 6. Sender Hardware Address (6 bytes for Ethernet): The MAC address of the device sending the message.
  • 7. Sender Protocol Address (4 bytes for IPv4): The IP address of the device sending the message.
  • 8. Target Hardware Address (6 bytes for Ethernet): The MAC address of the device being asked for. This field is empty (all zeros) in an ARP request.
  • 9. Target Protocol Address (4 bytes for IPv4): The IP address of the device being asked for.

Compare ARP and RARP in detail.

Feature ARP (Address Resolution Protocol) RARP (Reverse Address Resolution Protocol)
Main Job Maps an IP address to a MAC address. Maps a MAC address to an IP address.
Direction of Mapping From the Network layer to the Data Link layer. From the Data Link layer to the Network layer.
Request Type Broadcasts a request to the entire network. Broadcasts a request to a dedicated RARP server.
Used by All devices in a local network to find each other. Used by diskless machines to get an IP address when booting up.
Server Dependency Does not require a special server to work. Requires a dedicated RARP server to provide the IP addresses.
Table Maintained By The local host itself maintains its own ARP cache. The RARP server maintains a table of MAC-to-IP mappings.
Modern Usage Still widely used today. Largely replaced by modern protocols like DHCP and BOOTP.
